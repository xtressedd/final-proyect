Explicación del Código y Comandos

A continuación, detallo la estructura y configuración de los contenedores utilizados en mi proyecto, así como una explicación de cada componente definido en el archivo docker-compose.yml.

docker-compose.yml

Este archivo define cómo se ejecutarán los contenedores del proyecto, permitiendo crear una red en la que interactúan entre sí.

version: '3'

Estoy utilizando la versión 3 de Docker Compose, que establece las reglas y la sintaxis para definir los servicios del proyecto.

Servicios Definidos en docker-compose.yml

1. Servicio Suricata

  suricata:
    image: jasonish/suricata:latest
    container_name: suricata
    networks:
      - monitoring_network
    cap_add:
      - NET_ADMIN
      - NET_RAW
    volumes:
      - ./suricata_logs:/var/log/suricata
    ports:
      - "8080:8080"

image: jasonish/suricata:latest: Utilizo esta imagen de Docker basada en Suricata, una herramienta de análisis de seguridad de red.

container_name: suricata: El contenedor se llamará suricata.

networks: Se conectará a la red interna monitoring_network.

cap_add: Añadí permisos adicionales (NET_ADMIN y NET_RAW) para permitir el análisis de tráfico de red.

volumes: Los logs de Suricata se almacenarán en ./suricata_logs.

ports: Expongo el puerto 8080 para acceder al servicio.

2. Servicio de la Aplicación (Python)

  app:
    image: python:3.9-slim
    container_name: app
    networks:
      - monitoring_network
    volumes:
      - ./app:/app
    working_dir: /app
    command: python3 app.py
    ports:
      - "5000:5000"

image: python:3.9-slim: Uso una imagen ligera de Python 3.9 para la aplicación.

container_name: app: El contenedor de la aplicación se llama app.

volumes: Monté el directorio ./app dentro del contenedor.

working_dir: /app: Establecí /app como el directorio de trabajo.

command: python3 app.py: El contenedor ejecutará app.py.

ports: Expuse el puerto 5000 para acceder a la aplicación.

3. Servicio Elasticsearch

  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:7.10.0
    container_name: elasticsearch
    networks:
      - monitoring_network
    environment:
      - discovery.type=single-node
    ports:
      - "9200:9200"

image: Uso la imagen oficial de Elasticsearch.

environment: Configuré discovery.type=single-node para un entorno de pruebas.

ports: Expuse el puerto 9200 para Elasticsearch.

4. Servicio Kibana

  kibana:
    image: docker.elastic.co/kibana/kibana:7.10.0
    container_name: kibana
    networks:
      - monitoring_network
    environment:
      - ELASTICSEARCH_URL=http://elasticsearch:9200
    ports:
      - "5601:5601"

image: Uso la imagen oficial de Kibana.

environment: Configuré ELASTICSEARCH_URL para que Kibana pueda comunicarse con Elasticsearch.

ports: Expuse el puerto 5601 para la interfaz web de Kibana.

Redes Utilizadas

networks:
  monitoring_network:
    driver: bridge

monitoring_network: Definí esta red para permitir la comunicación entre los contenedores Suricata, la aplicación, Elasticsearch y Kibana.

Con esta configuración, logré integrar herramientas de análisis de tráfico de red y visualización de datos de seguridad dentro de un entorno Dockerizado.
